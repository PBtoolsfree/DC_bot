# Enterprise Web Dashboard Guide

The Management Bot Dashboard is a full-stack, enterprise-grade SaaS application designed to help you configure, monitor, and analyze your Discord communities at scale.

## Architecture

The dashboard is separated into a Next.js 15 App Router frontend and a FastAPI backend.
They live inside the `/dashboard` directory.
- `dashboard/frontend`: React 19, TailwindCSS, Shadcn UI, Next.js.
- `dashboard/backend`: FastAPI, SQLAlchemy Async, Redis WebSockets.

Both the Bot and the Backend share access to the same PostgreSQL database schemas.

## Authentication (Discord OAuth2)
We bypass traditional username/password flows. Users click "Login with Discord," which utilizes Discord OAuth2 to verify their identity.
A secure JWT is then generated by the FastAPI backend and used for subsequent API requests.

## Hybrid RBAC System
Unlike simple bots that rely entirely on Discord Server Roles for dashboard access, this dashboard uses a **Hybrid RBAC** (Role-Based Access Control) system.

- Server Owners automatically get full `OWNER` dashboard access.
- Owners can invite other Discord users to the dashboard and assign them roles: `ADMIN`, `MODERATOR`, or `VIEWER`.
- For extreme granularity, the `CUSTOM` role allows you to toggle specific permissions (e.g., "Can view analytics, but cannot change Anti-Nuke settings").

This system is backed by the `DashboardMember` table in the database.

## Live Events (WebSockets)
The `/logs` and `/security` pages on the dashboard connect to the FastAPI backend via WebSockets.
When an event occurs in Discord (e.g., an AutoMod violation), the bot pushes a payload to a Redis Pub/Sub channel. The FastAPI backend broadcasts this instantly to all connected dashboard clients for real-time visualization.

## Development

**Start Backend (FastAPI)**
```bash
cd dashboard/backend
uvicorn main:app --reload --port 8000
```

**Start Frontend (Next.js)**
```bash
cd dashboard/frontend
npm run dev
```

Next.js is configured with a proxy (`next.config.ts`) that automatically routes `/api/v1/*` requests to `localhost:8000`.
